# 测控补丁

西安交通大学测控专业特色资料与经验整理仓库。

GitHub 地址：[XJTU-Instrumentation-Share/XJTU-Measurement-and-Control-Share](https://github.com/XJTU-Instrumentation-Share/XJTU-Measurement-and-Control-Share)

一个面向测控专业特色内容的补丁包：哪里资料稀缺，哪里学习体验粗糙，哪里就尽量补一小块。

## 项目定位

西安交通大学的测控专业原本分为电测控、机测控、食品测控等方向，分散在电气学院、机械学院等学院。2023 年，学校在原测控专业基础上成立西安交通大学仪器科学与技术学院。

测控本身就是一个很“杂”的专业。它在培养方案里杂糅了电气、机械工程、自动化等方向的专业课，同时也有自己的特色内容，例如光学、误差理论、仪器设计、测试技术、智能仪器、光电检测等。相比主流公共基础课和专业基础课，这些课程的复习资料、经验帖和结构化信息往往更少，也更依赖一届一届同学之间的口口相传。

所以这个仓库叫“测控补丁”。

## 收录范围

本仓库主要但不限于收录以下内容：

- 从公开渠道、学长学姐或同学处搜集来的课程资料；
- 基于老师 PPT、课堂材料或教材，用 AI 工具整理后再人工校对的复习资料；
- 课程学习经验、复习建议、实验代码、实验文档、实验避坑、选课体验、培养方案理解；
- 测控特色课程相关的知识图谱、资料索引、学习路线和工具说明。

本仓库原则上只包含并接收测控专业特色资料，以避免重复劳动。其他通用课程资料已经有许多前人积累，例如 [XJTU-Share](https://github.com/cantjie/XJTU-Share)，请优先向这些更通用的仓库查找或投稿。

## 暂不收录

除非内容与测控特色课程直接相关，本仓库通常不收录：

- 高等数学、线性代数、概率论、大学物理、思政类课程等公共基础课资料；
- 模拟电子技术、数字电子技术、工程制图等已有大量成熟资料的专业基础课资料；
- 与测控专业无明显关系的其他专业课程资料；
- 毕业设计、与测控特色内容无关的课程设计成品、明显只用于直接提交的完整作业；
- 包含个人隐私、未授权传播内容或明显侵权风险的文件。

## 关于实验材料

本仓库允许收录实验代码、实验说明、实验文档、环境配置记录和必要的数据处理脚本。收录这些材料的目的，是帮助后来者理解实验原理、复现实验流程、排查环境问题，并在已有基础上继续优化。

请不要把仓库中的实验材料直接改名提交，也不要把它当作绕过课程训练的捷径。更合理的使用方式是：读懂它、运行它、对照自己的实验要求修改它、找出它的问题，再写出属于自己的版本。

在直接抄袭之前，请先想一想：这个项目是公开的。老师、助教、同学和后来者都可能看到同一份材料。把公开仓库里的内容原样提交，不仅风险很高，也违背了这个项目存在的初衷。

>西安交通大学学生违纪处分办法

>（五）在公开发表的学术论文、报告、设计中或在提交的学位论文、课程设计、毕业设计中，存在抄袭、篡改、伪造等学术不端行为，情节较轻且认识态度端正者，给予留校察看处分，情节严重者，给予开除学籍处分;
购买、出售或者组织买卖、由他人代写、替他人代写或组织代写学术论文、学位论文、课程设计、毕业设计者，给予开除学籍处分；

之所以选择公开收录部分实验材料，是因为 AI 已经足够强大，大家事实上也会使用 AI 辅助写代码、整理报告和排查问题。与其让每个人在不透明的信息差里各自摸索，不如把可以公开的基底材料放出来，让讨论、理解和改进发生在更清楚的地方。

## 推荐目录结构

每门课程建议单独建目录，并尽量按资料类型分类：

```text
课程名称/
  README.md
  复习资料/
  PPT整理/
  往年题/
  实验/
  经验/
  参考资料/
```

课程目录下的 `README.md` 建议写明：

- 课程名称、开课学院或方向；
- 适用届数或培养方案版本；
- 主要内容与考核方式；
- 推荐学习资料；
- 资料来源、整理者和更新时间；
- 需要特别注意的版权、隐私或使用限制。

## AI 整理说明

AI 工具正在改变大学教育，也正在改变资料整理方式。本仓库欢迎使用 AI 帮助整理课堂 PPT、教材笔记、复习提纲和知识框架，但请尽量遵守以下原则：

- 标注资料来源，例如课程名、学期、教师、教材或原始材料类型；
- 标注整理方式，例如“由 PPT 经 AI 初整理，人工校对”；
- 不要直接上传未获授权的完整课件、教材扫描件或付费资料；
- 对公式、定义、结论和考试范围进行人工核对；
- 明确区分“课堂原文”“个人理解”和“AI 生成整理”。

AI 可以做整理工具，但不应该替代判断。

## 投稿方式

欢迎通过 pull request 投稿。投稿前请先阅读 [CONTRIBUTING.md](CONTRIBUTING.md)。

如果你不熟悉 GitHub，也可以先通过 issue 说明你想补充的资料类型、课程名称和大致内容，再由维护者协助整理。

如果嫌 GitHub 投稿太麻烦，也可以直接联系 QQ：`1031804897` 或者 向skmail@stu.xjtu.edu.cn以附件形式投稿。我会在整理后标注投稿来源；如果你希望匿名，也请在投稿时说明。

投稿时请尤其注意：

- 保护自己和他人的姓名、学号、手机号、邮箱、宿舍号等个人信息；
- 实验代码和文档可以投稿，但请标注用途、来源和适用年份，不要鼓励他人原样提交；
- 尽量使用 PDF、Markdown、图片等便于预览的格式；
- 不上传压缩包，除非确有必要；
- 单个文件不要超过 GitHub 的普通文件大小限制，过大的文件请考虑 Git LFS 或外部链接。
